# Health — медицинская база

Личная медицинская база в Markdown для ведения пациентов с помощью AI-ассистента.

## Живой пример

🧬 **[Дашборд генетического профиля](https://rai220.github.io/giga-health-public/dashboard/)** — игровая визуализация фармакогенетического билда (образовательный пример, не медицинская рекомендация).

## Для чего

- Единый источник правды по диагнозам, лечению, обследованиям
- Отслеживание динамики во времени
- Подготовка к визитам к врачу
- AI-анализ медицинских данных и генетики

## Структура

```
health/
├── example_patient/       # Пример заполненного пациента
│   └── genetic/           # Расшифровки геномов (raw data)
├── drugs/                 # Справочник препаратов
├── tools/                 # Инструменты (анализ генетики и др.)
├── TEMPLATE/              # Шаблон для нового пациента
├── AGENTS.md              # Инструкции для AI-ассистента
└── README.md
```

Описание файлов пациента — в [AGENTS.md](AGENTS.md).

## Быстрый старт

### 1. Создать пациента

```bash
cp -r TEMPLATE/ имя_пациента/
```

### 2. Заполнить файлы

Минимум для старта: `BIO.md`, `SUMMARY.md`, `PROBLEMS.md`.

### 3. Использовать с AI

Откройте проект в Cursor / Claude Code. AI-ассистент автоматически подхватит инструкции из `AGENTS.md`.

## Соглашения

| Что | Формат |
|-----|--------|
| Даты | `YYYY-MM-DD`. Приблизительные — `YYYY` или `YYYY-MM` с пометкой `≈` |
| Источники | Указывать кто/откуда: врач, лаборатория, самонаблюдение |
| Статусы диагнозов | `active` / `resolved` / `suspected` / `ruled_out` |
| Повторяющиеся данные | Таблицы (лекарства, визиты, анализы) |
